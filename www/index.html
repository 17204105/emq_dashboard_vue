HTTP API

Introduction

Apart from this help page, all URIs will serve only resources of type application/json, and will require HTTP basic authentication. The default user is admin/admin.

The emqttd_dashboard plugin provides a web management console. The Dashboard helps monitor broker's running status, statistics and metrics of MQTT packets.

Reference

GET POST Path Description
Y Y /api/stats The Statistics of Clients, Sessions, Topics and Subscriptions
Y Y /api/nodes List of clustered nodes
Y Y /api/metrics The broker metrics
Y Y /api/listeners List of all TCP listeners
Y Y /api/clients List of all connected clients. support GET request like this:
http://localhost:18083/api/clients?curr_page=1&page_size=100&client_key=Key
Y Y /api/sessions List of all sessions. support GET request like this:
http://localhost:18083/api/sessions?curr_page=1&page_size=100&client_key=Key
Y Y /api/topics List of all Topics
Y Y /api/routes List of all routes. support GET request like this:
http://localhost:18083/api/routes?curr_page=1&page_size=100&topic=Topic
Y Y /api/subscriptions List of all Subscriptions. support GET request like this:
http://localhost:18083/api/subscriptions?curr_page=1&page_size=100&client_key=Key
Y Y /api/plugins Show loaded plugins
Y Y /api/users List of all admins
